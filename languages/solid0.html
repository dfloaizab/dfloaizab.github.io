Principios SOLID

Single Responsibility Principle

Una clase debe tener una única razón para cambiar.

Aplicación

class Reporte: def generar(self): pass

No aplicación

class Reporte: def generar(self): pass def guardar_db(self): pass

Beneficio: mantenimiento claro. Trade-off: más clases.

Open/Closed Principle

Abierto a extensión, cerrado a modificación.

class Descuento: def calcular(self, tipo): pass

Trade-off: mayor complejidad inicial.

Liskov Substitution Principle

Las subclases deben ser sustituibles por sus clases base.

class Ave: def volar(self): pass

Problema típico: pingüinos que no vuelan.

Interface Segregation Principle

No obligar a implementar métodos innecesarios.

class Impresora: def imprimir(self): pass

Dependency Inversion Principle

Depender de abstracciones, no de implementaciones.

class Servicio: def __init__(self, repo): self.repo = repo
