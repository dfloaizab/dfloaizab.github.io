Fundamentos de Programación Orientada a Objetos

1. Abstracción

La abstracción es el proceso de modelado mediante el cual se identifican las propiedades relevantes de una entidad, ignorando detalles de implementación. Constituye el fundamento para la construcción de sistemas complejos mediante capas de representación.

Diagrama conceptual

Ejemplo: sistema de pagos

from abc import ABC, abstractmethod class MetodoPago(ABC): @abstractmethod def procesar_pago(self, monto: float) -> str: pass class TarjetaCredito(MetodoPago): def procesar_pago(self, monto): return f"Procesando {monto} con tarjeta" class PayPal(MetodoPago): def procesar_pago(self, monto): return f"Procesando {monto} con PayPal"

La abstracción permite desacoplar el sistema del mecanismo específico de pago.

2. Encapsulamiento

El encapsulamiento controla el acceso al estado interno de los objetos, garantizando invariantes y evitando modificaciones no controladas.

Diagrama

Ejemplo realista

class CuentaBancaria: def __init__(self, saldo_inicial: float): self.__saldo = saldo_inicial def depositar(self, monto: float): if monto <= 0: raise ValueError("Monto inválido") self.__saldo += monto def retirar(self, monto: float): if monto > self.__saldo: raise ValueError("Fondos insuficientes") self.__saldo -= monto @property def saldo(self): return self.__saldo

Aquí se preserva el invariante: saldo >= 0.

3. Herencia

La herencia permite definir jerarquías donde clases derivadas extienden o especializan comportamientos.

Diagrama UML

Ejemplo

class Empleado: def calcular_salario(self): raise NotImplementedError class Ingeniero(Empleado): def calcular_salario(self): return 5000 class Gerente(Empleado): def calcular_salario(self): return 8000

Permite reutilización y especialización controlada.

4. Polimorfismo

El polimorfismo permite tratar objetos de distintas clases mediante una interfaz común. También, diversificar el comportamiento de las clases a través de la sobrecarga de funciones y operadores.

Ejemplo aplicado

def procesar_nomina(empleados): total = 0 for e in empleados: total += e.calcular_salario() return total

El algoritmo, en este caso, no depende del tipo concreto.
