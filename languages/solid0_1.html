Principios SOLID

SRP

Una clase debe tener una única razón de cambio. Esto reduce el acoplamiento y mejora la cohesión.

Correcto

class GeneradorReporte: def generar(self): pass class PersistenciaReporte: def guardar(self): pass

Incorrecto

class Reporte: def generar(self): pass def guardar(self): pass

Trade-off: proliferación de clases.

OCP: Open-Closed Principle: Open for extension, closed for modification

class Forma: def area(self): pass class Circulo(Forma): def area(self): return 3.14 * 2**2

No modificar código existente para agregar nuevas formas.

LSP: Liskov Substitution Principle

class Ave: def volar(self): pass class Pinguino(Ave): def volar(self): raise Exception("No puede volar")

Violación: rompe sustitución.

ISP

class Impresora: def imprimir(self): pass class Escaner: def escanear(self): pass

DIP

class Servicio: def __init__(self, repositorio): self.repo = repositorio

Uso de inyección de dependencias.
