Principios SOLID

SRP: Single Responsibility Principle

Una clase debe tener una única razón de cambio y un único papel o rol en el módulo. Esto permite generar alta cohesión.

class GeneradorFactura: def generar(self): pass class RepositorioFactura: def guardar(self): pass

Trade-off: aumento en número de clases.

OCP: Open-Closed Principle - Open for extension, Closed for modification

Las entidades deben poder ser extensibles sin necesidad de modificar su código.

class MetodoPago: def pagar(self): pass class Cripto(MetodoPago): def pagar(self): return "Pago cripto"

LSP: Liskov Substitution Principle

Las subclases deben ser sustituibles por su super-clase, o clase madre, sin alterar el comportamiento esperado.

class Ave: def volar(self): pass class Aguila(Ave): def volar(self): return "vuela" class Pinguino(Ave): def volar(self): raise Exception("Violación LSP")

ISP: Interface Segregation Principle

Si se diseña una jerarquía de clases para aplicar herencia y polimorfismo, no se debe obligar a una clase a implementar métodos que no va a usar. Cada interfaz ("contrato"), debe tener solo las funciones necesarias que definan su comportamiento y funcionalidad.

class Impresora: def imprimir(self): pass class Multifuncional(Impresora): def escanear(self): pass

DIP: Dependency Inversion Principle

Depender de abstracciones, no de clases concretas.

class Repositorio: def guardar(self): pass class Servicio: def __init__(self, repo: Repositorio): self.repo = repo

Una de las soluciones es la "inyección de dependencias": Se parametriza la dependencia de una abstracción en la funcionalidad de una clase determinada
