01 · Fundamentos

¿Qué es un Grafo?

Un grafo es una estructura matemática G = (V, E) compuesta por un conjunto de vértices (o nodos) V y un conjunto de aristas (o arcos) E ⊆ V × V que representan relaciones entre pares de vértices.

Los grafos son la estructura de datos más versátil en Ciencias de la Computación. Cualquier sistema que pueda modelarse como entidades con relaciones —redes sociales, rutas de transporte, dependencias de software, circuitos electrónicos, la propia Internet— es, en el fondo, un grafo.

Conceptos fundamentales

⬤

Vértice / Nodo

Entidad fundamental del grafo. Puede representar una ciudad, una persona, un servidor, un estado, etc.

━

Arista / Arco

Relación entre dos vértices. Puede ser con o sin dirección, y puede tener un peso asociado.

🔢

Grado (degree)

Número de aristas que inciden en un vértice. En dígrafos: grado de entrada y grado de salida.

📍

Camino (path)

Secuencia de vértices v₀, v₁, …, vₙ tal que cada par consecutivo está conectado por una arista.

🔄

Ciclo (cycle)

Camino cerrado donde el vértice inicial y el final son el mismo, sin repetir vértices intermedios.

🕸️

Componente conexa

Subgrafo donde existe un camino entre cualquier par de vértices. Un grafo puede tener varias.

02 · Historia

De Euler a la Web Moderna

La teoría de grafos nació de un acertijo geográfico y se convirtió en la base matemática de las redes computacionales del siglo XXI.

El problema de los puentes de Königsberg (1736)
1736

Leonhard Euler — El nacimiento de la teoría de grafos

Euler demuestra que es imposible cruzar los 7 puentes de Königsberg exactamente una vez. Abstrae el problema como un grafo y sienta las bases con el concepto de camino euleriano: un grafo tiene camino euleriano si y sólo si tiene 0 ó 2 vértices de grado impar.

1847

Gustav Kirchhoff — Teoría de redes eléctricas

Aplica grafos para analizar circuitos eléctricos. Sus leyes de voltaje y corriente son, esencialmente, propiedades de grafos.

1857 – 1878

Hamilton, Cayley y el problema de los 4 colores

Hamilton introduce los ciclos hamiltonianos. Cayley estudia árboles. Francis Guthrie formula el famoso problema de los 4 colores, que no sería probado hasta 1976.

1936

Dénes König — Primera monografía de teoría de grafos

Theorie der endlichen und unendlichen Graphen, el primer tratado sistemático de la disciplina.

1956 – 1959

Dijkstra, Prim, Kruskal — Algoritmos fundacionales

Edsger Dijkstra publica su famoso algoritmo de caminos más cortos. Prim y Kruskal publican algoritmos para árboles de expansión mínima, esenciales para redes.

1969 – hoy

Internet y la Web — El grafo más grande del mundo

ARPANET, el precursor de Internet, es modelado como grafo. Google nace de PageRank, un algoritmo sobre grafos. Hoy, redes sociales, sistemas de recomendación y IA son aplicaciones directas de la teoría de grafos.

Matriz de Adyacencia

Una matriz cuadrada A de tamaño n × n donde A[i][j] = w si existe una arista de peso w entre los vértices i y j, y 0 (o ∞) en caso contrario.

Matriz de adyacencia del grafo G (con pesos)

Los valores verdes son los pesos de las aristas. Ceros = sin conexión.

Implementación en Java

Matriz de AdyacenciaJava
public class GrafoMatriz {

    private int[][] adj;   // matriz de adyacencia
    private int  n;        // número de vértices

    public GrafoMatriz(int vertices) {
        this.n   = vertices;
        this.adj = new int[n][n]; // inicializa con 0
    }

    /** Agrega arista no dirigida con peso */
    public void agregarArista(int u, int v, int peso) {
        adj[u][v] = peso;
        adj[v][u] = peso; // eliminar si es dirigido
    }

    /** ¿Existe arista entre u y v? O(1) */
    public boolean existeArista(int u, int v) {
        return adj[u][v] != 0;
    }

    /** Vecinos de un vértice u — O(n) */
    public void imprimirVecinos(int u) {
        System.out.print("Vecinos de " + u + ": ");
        for (int v = 0; v < n; v++)
            if (adj[u][v] != 0)
                System.out.print(v + "(w="+adj[u][v]+") ");
        System.out.println();
    }

    public static void main(String[] args) {
        GrafoMatriz g = new GrafoMatriz(6);
        g.agregarArista(0, 1, 2);
        g.agregarArista(0, 2, 3);
        g.agregarArista(1, 3, 4);
        g.agregarArista(1, 4, 6);
        g.agregarArista(2, 4, 1);
        g.agregarArista(3, 5, 5);
        g.agregarArista(4, 5, 7);
        g.imprimirVecinos(1); // → Vecinos de 1: 0(w=2) 3(w=4) 4(w=6)
    }
}

Implementación en Python

Matriz de AdyacenciaPython
class GrafoMatriz:
    def __init__(self, vertices: int):
        self.n   = vertices
        self.adj = [[0] * vertices for _ in range(vertices)]

    def agregar_arista(self, u: int, v: int, peso: int = 1):
        self.adj[u][v] = peso
        self.adj[v][u] = peso   # omitir si es dirigido

    def existe_arista(self, u: int, v: int) -> bool:
        return self.adj[u][v] != 0

    def vecinos(self, u: int) -> list:
        return [(v, self.adj[u][v])
                for v in range(self.n)
                if self.adj[u][v] != 0]

# Uso:
g = GrafoMatriz(6)
g.agregar_arista(0, 1, 2)
g.agregar_arista(1, 3, 4)
print(g.vecinos(1))  # [(0, 2), (3, 4), (4, 6)]

Lista de Adyacencia

Un arreglo de listas (o diccionario) donde cada posición i contiene la lista de vecinos del vértice i. Es la representación preferida para grafos dispersos (sparse graphs), que son los más comunes en la práctica.

Lista de adyacencia del grafo G

Implementación en Java

Lista de AdyacenciaJava
import java.util.*;

public class GrafoLista {

    record Arista(int destino, int peso) {}

    private List<List<Arista>> adj;
    private int n;

    public GrafoLista(int vertices) {
        this.n   = vertices;
        this.adj = new ArrayList<>();
        for (int i = 0; i < n; i++)
            adj.add(new ArrayList<>());
    }

    public void agregarArista(int u, int v, int peso) {
        adj.get(u).add(new Arista(v, peso));
        adj.get(v).add(new Arista(u, peso));
    }

    /** BFS desde un origen */
    public void bfs(int origen) {
        boolean[] visitado = new boolean[n];
        Queue<Integer> cola = new LinkedList<>();
        visitado[origen] = true;
        cola.add(origen);

        while (!cola.isEmpty()) {
            int u = cola.poll();
            System.out.print(u + " ");
            for (Arista a : adj.get(u)) {
                if (!visitado[a.destino()]) {
                    visitado[a.destino()] = true;
                    cola.add(a.destino());
                }
            }
        }
    }

    /** DFS recursivo */
    public void dfs(int u, boolean[] visitado) {
        visitado[u] = true;
        System.out.print(u + " ");
        for (Arista a : adj.get(u))
            if (!visitado[a.destino()])
                dfs(a.destino(), visitado);
    }
}

Implementación en Python

Lista de AdyacenciaPython
from collections import defaultdict, deque

class GrafoLista:
    def __init__(self):
        # {vertice: [(vecino, peso), ...]}
        self.adj = defaultdict(list)

    def agregar_arista(self, u, v, peso=1):
        self.adj[u].append((v, peso))
        self.adj[v].append((u, peso))

    def bfs(self, origen) -> list:
        visitado, orden = {origen}, [origen]
        cola = deque([origen])
        while cola:
            u = cola.popleft()
            for v, _ in self.adj[u]:
                if v not in visitado:
                    visitado.add(v)
                    cola.append(v)
                    orden.append(v)
        return orden

    def dfs(self, origen) -> list:
        visitado, orden = set(), []
        def _dfs(u):
            visitado.add(u); orden.append(u)
            for v, _ in self.adj[u]:
                if v not in visitado: _dfs(v)
        _dfs(origen)
        return orden

# Uso:
g = GrafoLista()
g.agregar_arista(0, 1, 2)
g.agregar_arista(1, 3, 4)
print(g.bfs(0))  # [0, 1, 2, 3, 4, 5]

05 · Representaciones avanzadas

Otras Implementaciones

🗺️

Mapa de adyacencia (HashMap)

Reemplaza los arreglos por tablas hash: Map<Nodo, Map<Nodo, Peso>>. Ideal cuando los vértices son objetos arbitrarios (strings, IDs) y no enteros consecutivos.

📋

Lista de aristas (Edge List)

Almacena sólo las aristas: List<(u, v, peso)>. Mínimo espacio: O(E). Usada en algoritmos como Kruskal que iteran sobre todas las aristas.

🔢

CSR — Compressed Sparse Row

Representación eficiente para álgebra lineal y GPUs. Dos arreglos: uno con offsets de fila y otro con columnas de vecinos. Muy usado en redes sociales a escala masiva.

🧩

Matriz de incidencia

Matriz V × E donde M[v][e] = 1 si el vértice v incide en la arista e. Útil para análisis algebraico de grafos.

🗄️

Bases de datos de grafos

Neo4j, Amazon Neptune, ArangoDB. Almacenan grafos de millones de nodos con soporte nativo para Cypher / Gremlin. Usados en detección de fraude y redes sociales.

⚡

Representación implícita

El grafo no se almacena explícitamente; los vecinos se generan bajo demanda. Usada en búsqueda de estados (IA, puzzles, juegos) donde el grafo es potencialmente infinito.

06 · Aplicaciones

Grafos en la Práctica

Los grafos están presentes en prácticamente todas las áreas del software moderno. A continuación, las aplicaciones más relevantes para un ingeniero de sistemas.

🌐 Redes de Computadoras

Los routers son vértices y los enlaces son aristas ponderadas (por latencia o ancho de banda). Los protocolos de enrutamiento OSPF y BGP ejecutan algoritmos de grafos (Dijkstra, Bellman-Ford) en tiempo real.

👥 Redes Sociales

Facebook, LinkedIn e Instagram modelan usuarios como vértices y relaciones como aristas. PageRank, detección de comunidades y sistemas de recomendación son algoritmos de grafos aplicados a billones de nodos.

⚙️ Compiladores e IA

Los compiladores representan el flujo de control de un programa como un CFG (Control Flow Graph). Los modelos de deep learning son DAGs de operaciones. TensorFlow y PyTorch compilan y optimizan estos grafos.

🗺️ Sistemas GPS y Mapas

Google Maps y Waze modelan intersecciones como vértices y calles como aristas ponderadas por tiempo o distancia. Dijkstra y A* calculan la ruta óptima en milisegundos sobre grafos de millones de nodos.

🔗 Bases de datos de Grafos

Neo4j, Amazon Neptune. Permiten consultas de relaciones complejas que son ineficientes en SQL: detección de fraude (transacciones sospechosas), motores de recomendación, gestión de identidades.

🖥️ Sistemas Operativos

La detección de deadlock se realiza mediante un grafo de espera de recursos (RAG). Si el grafo tiene un ciclo, hay deadlock. La ordenación topológica de un DAG gestiona el orden de carga de módulos y dependencias.
