01 · Representación

Lista de Adyacencia

Una lista de adyacencia representa un grafo G = (V, E) como un arreglo de V listas encadenadas. Cada posición i contiene la lista de todos los vértices directamente conectados a i. Opcionalmente, cada nodo de la lista guarda también el peso de la arista.

Es la representación estándar de facto en programación competitiva y en implementaciones industriales de algoritmos sobre grafos, porque es eficiente en espacio para grafos dispersos (la gran mayoría en la práctica: redes viales, sociales, de dependencias) y permite iterar los vecinos en O(deg(v)) en lugar del O(V) que impone la matriz.

Variantes de la lista de adyacencia

📋

ArrayList / Vector (más común)

List<List<int[]>> en Java o vector<vector<pair<int,int>>> en C++. Acceso O(1) por índice del vértice. Sin peso: List<List<Integer>>.

🔗

Lista enlazada clásica

Nodos explícitos con puntero siguiente. Inserción O(1) al frente, pero peor localidad de caché que el ArrayList. Más usada en libros de texto que en competencia.

🗺️

HashMap de HashSets

Map<T, Set<T>>. Ideal cuando los vértices son objetos (strings, coordenadas). Soporta vértices no consecutivos y permite verificar si existe arista en O(1) amortizado.

⚡

CSR — Compressed Sparse Row

Dos arreglos planos: offset[] y dest[]. Máxima eficiencia de caché. Usada en bibliotecas de grafos de alto rendimiento (GraphX, NetworkX interno).

02 · Comparativa

Lista vs Matriz de Adyacencia

Ninguna representación es universalmente superior. La elección depende del tipo de grafo (denso vs. disperso) y de las operaciones que más se realizan.

Operación / Propiedad Lista de Adyacencia Matriz de Adyacencia
Espacio O(V + E) — sólo almacena aristas existentes O(V²) — toda la cuadrícula aunque esté vacía
¿Existe arista (u, v)? O(deg(u)) — hay que recorrer la lista de u O(1) — acceso directo A[u][v]
Agregar arista O(1) amortizado — agregar al final de la lista O(1) — asignar A[u][v] = w
Eliminar arista O(deg(u)) — buscar y remover de la lista O(1) — poner A[u][v] = 0
Recorrer vecinos de u O(deg(u)) — iterar sólo los vecinos reales O(V) — revisar todas las columnas de la fila
BFS / DFS O(V + E) — óptimo para grafos dispersos O(V²) — inevitable al recorrer cada fila
Dijkstra (con heap) O((V+E) log V) O(V²) sin heap / O(V² log V) con heap
Floyd-Warshall Se convierte internamente a matriz — no aplica O(V³) — nativamente eficiente sobre matriz
Localidad de caché Media — punteros dispersos en memoria Alta — memoria contigua, prefetch amigable
Grafos densos (E ≈ V²) No óptima — espacio similar, overhead de punteros Preferida — simple y cache-friendly
Grafos dispersos (E << V²) Preferida — ahorra memoria y tiempo de traversal Ineficiente — 99% de la matriz puede ser 0
Multiaristas / multigrafos Soportado — múltiples entradas del mismo vecino No soportado — A[u][v] es un único valor
Caso de uso típico Redes sociales, mapas, web graph, compiladores Grafos pequeños, Floyd-Warshall, álgebra lineal
Regla práctica: Si E < V² / 10 (el grafo tiene menos del 10% de las aristas posibles) usa lista de adyacencia. Si el grafo es casi completo o V es pequeño (V ≤ 1000), la matriz puede ser más conveniente por su simplicidad de acceso. En programación competitiva se prefiere casi siempre la lista.

Código en Java

Clase Grafo — lista de adyacenciaJava 17
import java.util.*;

public class Grafo {

    // ── Nodo interno para representar una arista ────────────────
    record Arista(int destino, int peso) {
        @Override
        public String toString() {
            return destino + "(w=" + peso + ")";
        }
    }

    // ── Campos principales ──────────────────────────────────────
    private final int V;              // número de vértices
    private final boolean dirigido;   // ¿es dirigido?
    private final List<List<Arista>> adj; // lista de adyacencia
    private int E;                     // número de aristas

    // ── Constructor ─────────────────────────────────────────────
    public Grafo(int vertices, boolean dirigido) {
        this.V = vertices;
        this.dirigido = dirigido;
        this.E = 0;
        this.adj = new ArrayList<>();
        for (int i = 0; i < V; i++)
            adj.add(new ArrayList<>());
    }

    // ── Agregar arista (con peso) ────────────────────────────────
    public void agregarArista(int u, int v, int peso) {
        validar(u); validar(v);
        adj.get(u).add(new Arista(v, peso));
        if (!dirigido)
            adj.get(v).add(new Arista(u, peso)); // ← bidireccional
        E++;
    }

    // Sobrecarga sin peso (peso = 1 por defecto)
    public void agregarArista(int u, int v) { agregarArista(u, v, 1); }

    // ── Vecinos de un vértice ────────────────────────────────────
    public List<Arista> vecinos(int u) { return adj.get(u); }

    // ── ¿Existe arista (u, v)? ──────────────────────────────────
    public boolean existeArista(int u, int v) {
        return adj.get(u).stream()
                   .anyMatch(a -> a.destino() == v);
    }

    // ── Grado de un vértice ─────────────────────────────────────
    public int grado(int u) { return adj.get(u).size(); }

    // ── Imprimir la lista completa ───────────────────────────────
    public void imprimir() {
        System.out.println("Grafo (" + V + " vértices, " + E + " aristas):");
        for (int u = 0; u < V; u++) {
            System.out.print("  [" + u + "] → ");
            adj.get(u).forEach(a -> System.out.print(a + " "));
            System.out.println();
        }
    }

    // ── Validación ───────────────────────────────────────────────
    private void validar(int v) {
        if (v < 0 || v >= V)
            throw new IllegalArgumentException("Vértice "+v+" fuera de rango");
    }

    public int V() { return V; }
    public int E() { return E; }
}
BFS y DFS sobre GrafoJava 17
import java.util.*;

public class Recorridos {

    // ══════════════════════════════════════════════════
    //  BFS — Breadth-First Search (búsqueda por anchura)
    //  Usa una COLA (Queue) — FIFO
    // ══════════════════════════════════════════════════
    public static List<Integer> bfs(Grafo g, int origen) {
        boolean[] visitado = new boolean[g.V()];
        List<Integer> orden = new ArrayList<>();
        Queue<Integer> cola = new LinkedList<>();

        visitado[origen] = true;
        cola.offer(origen);

        while (!cola.isEmpty()) {
            int u = cola.poll();          // sacar el frente
            orden.add(u);

            // encolar todos los vecinos no visitados
            for (Grafo.Arista a : g.vecinos(u)) {
                if (!visitado[a.destino()]) {
                    visitado[a.destino()] = true;
                    cola.offer(a.destino());
                }
            }
        }
        return orden;
    }

    // ══════════════════════════════════════════════════
    //  DFS — Depth-First Search (búsqueda por profundidad)
    //  Usa la PILA de llamadas (recursión) o un Stack
    // ══════════════════════════════════════════════════
    public static List<Integer> dfs(Grafo g, int origen) {
        boolean[] visitado = new boolean[g.V()];
        List<Integer> orden = new ArrayList<>();
        dfsHelper(g, origen, visitado, orden);
        return orden;
    }

    private static void dfsHelper(Grafo g, int u,
                                    boolean[] visitado,
                                    List<Integer> orden) {
        visitado[u] = true;
        orden.add(u);
        for (Grafo.Arista a : g.vecinos(u)) {
            if (!visitado[a.destino()])
                dfsHelper(g, a.destino(), visitado, orden);
        }
        // ← al salir de la recursión: post-order disponible aquí
    }

    // ── BFS con distancias desde el origen ──────────────────────
    public static int[] bfsDistancias(Grafo g, int origen) {
        int[] dist = new int[g.V()];
        Arrays.fill(dist, -1);               // -1 = no alcanzado
        dist[origen] = 0;
        Queue<Integer> cola = new LinkedList<>();
        cola.offer(origen);

        while (!cola.isEmpty()) {
            int u = cola.poll();
            for (Grafo.Arista a : g.vecinos(u)) {
                if (dist[a.destino()] == -1) {
                    dist[a.destino()] = dist[u] + 1;
                    cola.offer(a.destino());
                }
            }
        }
        return dist; // dist[v] = saltos mínimos desde origen a v
    }

    // ── DFS para detectar ciclos en grafo dirigido ───────────────
    public static boolean tieneCiclo(Grafo g) {
        boolean[] visitado  = new boolean[g.V()];
        boolean[] enPila    = new boolean[g.V()];
        for (int v = 0; v < g.V(); v++)
            if (!visitado[v] && dfsCiclo(g, v, visitado, enPila))
                return true;
        return false;
    }

    private static boolean dfsCiclo(Grafo g, int u,
                                       boolean[] vis, boolean[] pila) {
        vis[u] = pila[u] = true;
        for (Grafo.Arista a : g.vecinos(u)) {
            if (!vis[a.destino()] && dfsCiclo(g, a.destino(), vis, pila))
                return true;
            if (pila[a.destino()]) return true; // arista de retroceso
        }
        pila[u] = false;
        return false;
    }

    // ── Demo principal ───────────────────────────────────────────
    public static void main(String[] args) {
        Grafo g = new Grafo(6, false);
        g.agregarArista(0, 1, 4); g.agregarArista(0, 2, 7);
        g.agregarArista(1, 3, 2); g.agregarArista(1, 4, 9);
        g.agregarArista(2, 4, 3); g.agregarArista(3, 5, 5);
        g.agregarArista(4, 5, 6);

        g.imprimir();
        System.out.println("BFS desde 0: " + bfs(g, 0));
        // → [0, 1, 2, 3, 4, 5]
        System.out.println("DFS desde 0: " + dfs(g, 0));
        // → [0, 1, 3, 5, 4, 2]
        System.out.println("Distancias BFS desde 0: "
            + Arrays.toString(bfsDistancias(g, 0)));
        // → [0, 1, 1, 2, 2, 3]
    }
}
04 · Algoritmo

BFS — Búsqueda por Anchura

BFS (Breadth-First Search) explora el grafo nivel a nivel: primero todos los vecinos directos del origen, luego los vecinos de los vecinos, y así sucesivamente. Usa una cola (FIFO) como estructura auxiliar y garantiza encontrar el camino mínimo en grafos no ponderados.

Traza visual paso a paso

BFS desde vértice 0 — el grafo de referencia
BFS
Breadth-First Search
Complejidad: O(V + E)

Estructura clave: Cola FIFO — los vértices se procesan en el orden en que son descubiertos.

  • Camino mínimo en grafos no ponderados (distancia en saltos)
  • Detección de bipartición — 2-coloración del grafo
  • Componentes conexas en grafos no dirigidos
  • Rastreadores web — explorar la web por niveles desde una URL
  • Social BFS — "personas a N grados de separación"
  • Broadcasting en redes — propagación de mensajes capa a capa
  • Puzzle 0–1 BFS con deque para aristas de peso 0 y 1
DFS
Depth-First Search
Complejidad: O(V + E)

Estructura clave: Pila implícita (recursión) o explícita — va tan profundo como puede antes de retroceder.

  • Detección de ciclos — aristas de retroceso (back edges)
  • Ordenación topológica de DAGs (compiladores, npm)
  • Componentes fuertemente conexas (SCC) — Kosaraju, Tarjan
  • Resolución de laberintos y puzzles con backtracking
  • Árbol generador de un grafo
  • Puntos de articulación y puentes (vulnerabilidad de redes)
  • Flood fill — relleno de regiones en grillas 2D

05 · DFS Visual

DFS — Traza Paso a Paso

DFS desde vértice 0 — árbol DFS y aristas de retroceso
Observación clave: En el árbol DFS, las aristas discontinuas amarillas son aristas de retroceso (back edges) — conectan un vértice con un ancestro ya visitado. Su existencia es la evidencia de un ciclo en el grafo. En grafos no dirigidos, toda arista que no es de árbol es de retroceso.

DFS iterativo — evita stackoverflow en grafos grandesJava
public static List<Integer> dfsIterativo(Grafo g, int origen) {
    boolean[] visitado = new boolean[g.V()];
    List<Integer> orden = new ArrayList<>();
    Deque<Integer> pila = new ArrayDeque<>();

    pila.push(origen);

    while (!pila.isEmpty()) {
        int u = pila.pop();           // sacar el tope
        if (visitado[u]) continue;   // puede estar duplicado
        visitado[u] = true;
        orden.add(u);

        // empujar vecinos en orden INVERSO para mantener
        // el mismo orden que la versión recursiva
        List<Grafo.Arista> vecinos = g.vecinos(u);
        for (int i = vecinos.size() - 1; i >= 0; i--)
            if (!visitado[vecinos.get(i).destino()])
                pila.push(vecinos.get(i).destino());
    }
    return orden;
}

// Ordenación topológica con DFS (para DAGs)
// Resultado: orden inverso de finalización
public static List<Integer> ordenTopologico(Grafo g) {
    boolean[] visitado = new boolean[g.V()];
    Deque<Integer> resultado = new ArrayDeque<>();

    for (int v = 0; v < g.V(); v++)
        if (!visitado[v])
            dfsTopoHelper(g, v, visitado, resultado);

    return new ArrayList<>(resultado); // frente = primero en orden
}

private static void dfsTopoHelper(Grafo g, int u,
                                       boolean[] vis,
                                       Deque<Integer> res) {
    vis[u] = true;
    for (Grafo.Arista a : g.vecinos(u))
        if (!vis[a.destino()])
            dfsTopoHelper(g, a.destino(), vis, res);
    res.push(u);  // ← añadir DESPUÉS de explorar todos los vecinos
}
06 · Aplicaciones Prácticas

Dos Ejemplos Concretos

Ejemplo 1 · BFS — Red de metro

¿Cuántas estaciones hay entre A y B?

Una red de metro se modela como grafo no ponderado: estaciones = vértices, líneas directas = aristas. BFS desde la estación A encuentra la ruta con menos transbordos hacia cualquier destino.

Red simplificada (5 estaciones)

¿Por qué BFS? Explora por niveles = por número de paradas. La primera vez que llega a F es garantizadamente el camino con menos transbordos. DFS podría encontrar A→C→E→F (también 3) pero podría también explorar primero A→C→... si hubiera una rama más larga.

Ejemplo 2 · DFS — Detección de ciclos en build

¿Hay dependencias circulares en el proyecto?

Un sistema de build (Maven, Gradle) tiene módulos como vértices y dependencias como aristas dirigidas. Si hay un ciclo (A depende de B, B depende de A), el build falla. DFS detecta ciclos con aristas de retroceso.

Grafo de dependencias con ciclo detectado

¿Por qué DFS? Durante el DFS se mantiene un conjunto de nodos "en la pila de llamadas". Si al explorar un vecino se encuentra uno que ya está en la pila → hay un ciclo. BFS no puede detectar ciclos de esta forma porque no razona sobre la profundidad de la exploración.
