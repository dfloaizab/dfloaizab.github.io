Persistencia en Java

Guardar el estado de una aplicación más allá de su ciclo de vida en memoria.

La persistencia es la capacidad de un programa para almacenar datos de forma que sobrevivan al cierre de la aplicación. En Java existen múltiples mecanismos con distintos niveles de complejidad, desde simples archivos de texto hasta bases de datos relacionales y sistemas ORM.

Niveles de persistencia

Se puede clasificar la persistencia en tres grandes niveles según la durabilidad, complejidad y escala de los datos:

N1

Persistencia básica

Archivos planos (.txt, .csv), archivos de propiedades (.properties) y serialización de objetos. Ideal para configuraciones, prototipos y datos simples de una sola aplicación.

N2

Persistencia estructurada

Bases de datos relacionales (MySQL, PostgreSQL) accedidas con JDBC. Soporta múltiples usuarios, transacciones e integridad referencial.

N3

Persistencia avanzada / ORM

Frameworks como JPA/Hibernate, Spring Data. Mapean objetos Java a tablas automáticamente. Adecuados para aplicaciones empresariales a gran escala.

ℹ️

En este semestre nos enfocamos en el Nivel 1: archivos planos, Properties y serialización. Estos mecanismos no requieren instalar servidores de base de datos y permiten aprender los conceptos esenciales de I/O en Java.

Archivos planos (Text I/O)

Un archivo plano almacena información como texto legible (caracteres Unicode). Java proporciona clases en java.io para escribir y leer estos archivos. Las clases más utilizadas en segundo semestre son FileWriter, BufferedWriter, FileReader y BufferedReader.

✅ Beneficios

Simplicidad, portabilidad, legibilidad humana directa, no requiere dependencias externas ni instalación.

📌 Cuándo usarlo

Logs de aplicación, exportación a CSV, listas de datos pequeñas, archivos de salida para reportes.

⚠️ Limitaciones

No tiene estructura de consulta, es vulnerable a errores de formato y poco eficiente para grandes volúmenes.

Ejemplo: Guardar y leer lista de estudiantes

GestorEstudiantes.java Java 
import java.io.*;
import java.util.*;

public class GestorEstudiantes {

    private static final String ARCHIVO = "estudiantes.txt";

    // Escribe una lista de estudiantes en el archivo (una por línea)
    public static void guardar(List<String> estudiantes) {
        try (BufferedWriter bw = new BufferedWriter(new FileWriter(ARCHIVO))) {
            for (String nombre : estudiantes) {
                bw.write(nombre);
                bw.newLine();   // separador de registro
            }
            System.out.println("✔ Estudiantes guardados en " + ARCHIVO);
        } catch (IOException e) {
            System.err.println("Error al escribir: " + e.getMessage());
        }
    }

    // Lee y devuelve la lista almacenada
    public static List<String> cargar() {
        List<String> lista = new ArrayList<>();
        File f = new File(ARCHIVO);
        if (!f.exists()) return lista;   // archivo no existe aún

        try (BufferedReader br = new BufferedReader(new FileReader(f))) {
            String linea;
            while ((linea = br.readLine()) != null) {
                if (!linea.isBlank()) lista.add(linea.trim());
            }
        } catch (IOException e) {
            System.err.println("Error al leer: " + e.getMessage());
        }
        return lista;
    }

    public static void main(String[] args) {
        List<String> lista = Arrays.asList("Ana Torres", "Carlos Ríos", "Lucía Paz");
        guardar(lista);

        List<String> recuperados = cargar();
        recuperados.forEach(System.out::println);
    }
}

Archivos de propiedades (.properties)

La clase java.util.Properties permite leer y escribir pares clave=valor en archivos .properties. Es el mecanismo estándar de Java para almacenar configuración de aplicaciones: URLs, credenciales, parámetros de negocio, mensajes internacionalizados, etc.

✅ Beneficios

Separación de la configuración del código fuente. Fácil de editar sin recompilar. Soporta internacionalización (i18n). Formato estandarizado y legible.

📌 Cuándo usarlo

Configuración de conexiones a BD, idiomas de la UI, rutas de directorios, parámetros de despliegue, mensajes de error personalizados.

Clase Java: LectorConfig.java

LectorConfig.java Java 
import java.io.*;
import java.util.Properties;

public class LectorConfig {

    private Properties props = new Properties();

    /** Carga el archivo de propiedades desde el classpath */
    public void cargar(String rutaArchivo) throws IOException {
        try (InputStream is = new FileInputStream(rutaArchivo)) {
            props.load(is);
        }
    }

    /** Obtiene un valor; retorna valorDefecto si la clave no existe */
    public String get(String clave, String valorDefecto) {
        return props.getProperty(clave, valorDefecto);
    }

    /** Actualiza un valor y persiste los cambios */
    public void set(String clave, String valor, String rutaArchivo) throws IOException {
        props.setProperty(clave, valor);
        try (OutputStream os = new FileOutputStream(rutaArchivo)) {
            props.store(os, "Configuración actualizada automáticamente");
        }
    }

    public static void main(String[] args) throws IOException {
        LectorConfig cfg = new LectorConfig();
        cfg.cargar("config.properties");

        String app  = cfg.get("app.nombre", "App");
        String host = cfg.get("db.host", "localhost");
        System.out.printf("Aplicación: %s | BD en: %s%n", app, host);

        // Cambiar el idioma y guardar
        cfg.set("ui.idioma", "en_US", "config.properties");
    }
}

Serialización de objetos

La serialización convierte un objeto Java completo (incluida su estructura interna) en una secuencia de bytes que se almacena en un archivo binario (.dat). La deserialización es el proceso inverso: reconstruye el objeto desde esos bytes.

Para serializar una clase, esta debe implementar la interfaz java.io.Serializable (marcador, sin métodos). Se recomienda declarar un campo serialVersionUID para control de versiones.

✅ Beneficios

Persiste el objeto completo con todos sus campos sin necesidad de mapeo manual. Muy práctico para guardar el estado de la aplicación.

📌 Cuándo usarlo

Sesiones de usuario, caché local, juegos (guardar partida), configuraciones de objetos complejos, transferencia de objetos en red (RMI).

⚠️ Limitaciones

El archivo binario no es legible por humanos. Cambios en la clase pueden invalidar archivos antiguos. No recomendado para interoperabilidad entre sistemas.

Ejemplo: Serializar y recuperar un objeto Estudiante

Estudiante.java Java 
import java.io.Serializable;

public class Estudiante implements Serializable {

    // Identificador de versión — evita incompatibilidades entre versiones de la clase
    private static final long serialVersionUID = 1L;

    private String  nombre;
    private int     codigo;
    private double  promedio;

    public Estudiante(String nombre, int codigo, double promedio) {
        this.nombre   = nombre;
        this.codigo   = codigo;
        this.promedio = promedio;
    }

    @Override
    public String toString() {
        return String.format("[%d] %s — Promedio: %.1f", codigo, nombre, promedio);
    }

    // Getters...
}
GestorSerializacion.java Java 
import java.io.*;

public class GestorSerializacion {

    private static final String ARCHIVO = "estudiante.dat";

    /** Serializa (guarda) el objeto en un archivo binario */
    public static void guardar(Estudiante e) {
        try (ObjectOutputStream oos =
                new ObjectOutputStream(new FileOutputStream(ARCHIVO))) {
            oos.writeObject(e);
            System.out.println("Objeto serializado correctamente.");
        } catch (IOException ex) {
            ex.printStackTrace();
        }
    }

    /** Deserializa (recupera) el objeto desde el archivo */
    public static Estudiante cargar() {
        try (ObjectInputStream ois =
                new ObjectInputStream(new FileInputStream(ARCHIVO))) {
            return (Estudiante) ois.readObject();
        } catch (IOException | ClassNotFoundException ex) {
            ex.printStackTrace();
            return null;
        }
    }

    public static void main(String[] args) {
        Estudiante e1 = new Estudiante("Valentina Gómez", 2023001, 4.2);
        guardar(e1);

        Estudiante recuperado = cargar();
        System.out.println("Recuperado → " + recuperado);
    }
}

Uso de Excepciones

Mecanismo de Java para detectar, propagar y manejar errores en tiempo de ejecución de forma estructurada.

Una excepción es un evento que interrumpe el flujo normal del programa. Java separa la lógica de negocio del manejo de errores mediante un sistema de excepciones basado en clases. Esto hace el código más robusto, legible y mantenible.

Jerarquía de excepciones

java.lang.Throwable
├── Error ← Errores graves del sistema (no capturar)
│ ├── OutOfMemoryError
│ └── StackOverflowError
└── Exception
    ├── IOException ← Checked: obligatorio manejarlas
    │ ├── FileNotFoundException
    │ └── EOFException
    ├── SQLException ← Checked
    ├── ClassNotFoundException ← Checked
    └── RuntimeException ← Unchecked: opcionales de capturar
        ├── NullPointerException
        ├── ArrayIndexOutOfBoundsException
        ├── NumberFormatException
        ├── ArithmeticException
        └── IllegalArgumentException
🔵 Checked Exceptions

El compilador obliga a manejarlas con try-catch o declararlas con throws. Representan situaciones esperadas pero anormales (ej: archivo no encontrado, error de red).

🟡 Unchecked Exceptions

Son subclases de RuntimeException. El compilador no las exige manejar. Generalmente indican errores de programación (null pointer, índice inválido, división por cero).

try / catch / finally / throws / throw

Palabra claveUsoDescripción
tryBloque protegidoContiene el código que puede lanzar una excepción.
catchCaptura y manejaEjecuta código de recuperación si ocurre la excepción indicada.
finallySiempre se ejecutaSe ejecuta haya o no excepción. Ideal para cerrar recursos.
throwsDeclaraciónDeclara que un método puede lanzar una checked exception (responsabilidad al llamador).
throwLanzamientoLanza explícitamente una excepción desde código.

Ejemplo completo: manejo de excepciones múltiples

CalculadoraSegura.java Java 
public class CalculadoraSegura {

    /**
     * Divide dos enteros con manejo robusto de errores.
     * @throws ArithmeticException  si el divisor es cero
     */
    public static double dividir(int a, int b) {
        if (b == 0) {
            throw new ArithmeticException("División por cero: el divisor no puede ser 0");
        }
        return (double) a / b;
    }

    /** Convierte un String a entero de forma segura */
    public static int parsearEntero(String texto) {
        try {
            return Integer.parseInt(texto);
        } catch (NumberFormatException e) {
            System.err.println("Valor no numérico: '" + texto + "' → se asume 0");
            return 0;
        }
    }

    public static void main(String[] args) {
        // ── 1. Multi-catch (Java 7+) ──────────────────────────
        try {
            String[] arreglo = new String[3];
            System.out.println(arreglo[5]);  // índice inválido
        } catch (ArrayIndexOutOfBoundsException | NullPointerException e) {
            System.err.println("Error de arreglo o nulo: " + e.getMessage());
        } finally {
            System.out.println("Bloque finally — siempre se ejecuta");
        }

        // ── 2. Captura de excepción lanzada manualmente ───────
        try {
            double resultado = dividir(10, 0);
        } catch (ArithmeticException e) {
            System.err.println("⚠ " + e.getMessage());
        }

        // ── 3. Conversión segura ──────────────────────────────
        int num = parsearEntero("abc");  // retornará 0
        System.out.println("Número parseado: " + num);
    }
}

Excepciones personalizadas

Se pueden crear excepciones propias extendiendo Exception (checked) o RuntimeException (unchecked) para representar errores específicos del dominio de la aplicación.

NotaInvalidaException.java Java 
/** Excepción de dominio: nota fuera del rango 0.0 – 5.0 */
public class NotaInvalidaException extends Exception {

    private final double notaRecibida;

    public NotaInvalidaException(double nota) {
        super("Nota inválida: " + nota + ". El rango permitido es [0.0, 5.0]");
        this.notaRecibida = nota;
    }

    public double getNotaRecibida() { return notaRecibida; }
}

// ── Uso en otra clase ─────────────────────────────────────
public class RegistroNotas {

    public void registrarNota(String estudiante, double nota)
            throws NotaInvalidaException {
        if (nota < 0.0 || nota > 5.0) {
            throw new NotaInvalidaException(nota);
        }
        System.out.printf("✔ Nota %.1f registrada para %s%n", nota, estudiante);
    }

    public static void main(String[] args) {
        RegistroNotas rn = new RegistroNotas();
        try {
            rn.registrarNota("Juan Pérez", 4.5);   // ✔
            rn.registrarNota("María Leal", 6.0);   // ✗ lanza excepción
        } catch (NotaInvalidaException e) {
            System.err.println("Error: " + e.getMessage());
        }
    }
}
✅

Buenas prácticas: captura siempre la excepción más específica primero; no captures Exception genéricamente si puedes evitarlo; nunca dejes un bloque catch vacío; usa el mensaje de excepción para dar contexto útil al desarrollador.

Interfaces de Usuario: AWT y Swing

Construcción de ventanas, formularios y controles gráficos en aplicaciones de escritorio con Java.

AWT · Abstract Window Toolkit

Primera librería gráfica de Java (java.awt.*). Los componentes son pesados: delegan el renderizado al sistema operativo. Limitados en personalización y aspecto visual.

Swing · javax.swing.*

Construido sobre AWT, pero con componentes ligeros dibujados 100% por Java. Mayor portabilidad, más componentes y aspecto unificado. Es el estándar para aplicaciones de escritorio en este semestre.

Contenedores principales

Un contenedor es un componente que puede albergar otros componentes (botones, campos de texto, paneles, etc.). Forman una jerarquía: el contenedor raíz es siempre un JFrame.

🖼 JFrame — Ventana principal

Representa la ventana del sistema operativo con barra de título, bordes y botones de sistema. Es el punto de entrada de la interfaz gráfica. Contiene un ContentPane donde se agregan los demás componentes.

📦 JPanel — Panel de agrupación

Contenedor genérico y liviano. Se usa para organizar subgrupos de componentes dentro del JFrame u otros paneles. Puede tener su propio LayoutManager y fondo.

💬 JDialog — Cuadro de diálogo

Ventana secundaria modal o no modal, hija de un JFrame. Útil para formularios de confirmación, mensajes de error, entradas de datos auxiliares.

📜 JScrollPane — Área con scroll

Contenedor que agrega barras de desplazamiento a cualquier componente (listas, tablas, áreas de texto) cuando su contenido supera el espacio visible.

EstructuraJFrame.java Java 
import javax.swing.*;
import java.awt.*;

public class EstructuraJFrame {
    public static void main(String[] args) {
        // Crear ventana principal
        JFrame ventana = new JFrame("Mi Primera Ventana");
        ventana.setSize(400, 300);
        ventana.setLocationRelativeTo(null);            // centrar en pantalla
        ventana.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);

        // Crear panel y agregar al ContentPane
        JPanel panel = new JPanel();
        panel.setBackground(Color.WHITE);
        panel.add(new JLabel("¡Hola, Swing!"));

        ventana.getContentPane().add(panel);
        ventana.setVisible(true);
    }
}

import javax.swing.*; import java.awt.*;

public class EjemploLayouts {
    public static void demo() {
        JFrame f = new JFrame(); f.setSize(500, 400);

        // ── BorderLayout (layout del JFrame por defecto) ──────
        f.setLayout(new BorderLayout(5, 5));
        f.add(new JButton("Menú"),    BorderLayout.NORTH);
        f.add(new JButton("Opciones"), BorderLayout.WEST);
        f.add(new JTextArea(),         BorderLayout.CENTER);
        f.add(new JButton("Estado"),   BorderLayout.SOUTH);

        // ── GridLayout para teclado numérico ──────────────────
        JPanel teclado = new JPanel(new GridLayout(4, 3, 4, 4));
        String[] teclas = {"7","8","9","4","5","6","1","2","3","0","C","="};
        for (String t : teclas) teclado.add(new JButton(t));

        // ── BoxLayout para formulario vertical ───────────────
        JPanel formulario = new JPanel();
        formulario.setLayout(new BoxLayout(formulario, BoxLayout.Y_AXIS));
        formulario.add(new JLabel("Nombre:"));
        formulario.add(new JTextField(20));
        formulario.add(Box.createVerticalStrut(10));  // espacio fijo
        formulario.add(new JButton("Guardar"));
    }
}

Código fuente:

VentanaRegistro.java Java 
import javax.swing.*;
import java.awt.*;

public class VentanaRegistro extends JFrame {

    // ── Declaración de componentes ───────────────────────────────────
    private JTextField  txNombre, txCodigo, txCorreo;
    private JComboBox<String> cbSemestre;
    private JButton     btnRegistrar, btnLimpiar;
    private JLabel      lblEstado;

    public VentanaRegistro() {
        initUI();
        registrarEventos();
    }

    private void initUI() {
        setTitle("🎓 Registro de Estudiante");
        setSize(420, 300);
        setLocationRelativeTo(null);
        setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
        setLayout(new BorderLayout(8, 8));

        // ── Panel del formulario (GridLayout 4 filas × 2 cols) ───────
        JPanel pnlForm = new JPanel(new GridLayout(4, 2, 6, 6));
        pnlForm.setBorder(BorderFactory.createTitledBorder("Datos del estudiante"));

        txNombre   = new JTextField();
        txCodigo   = new JTextField();
        txCorreo   = new JTextField();
        cbSemestre = new JComboBox<>(new String[]{
            "Primero (I)", "Segundo (II)", "Tercero (III)", "Cuarto (IV)"
        });

        pnlForm.add(new JLabel("Nombre:"));    pnlForm.add(txNombre);
        pnlForm.add(new JLabel("Código:"));    pnlForm.add(txCodigo);
        pnlForm.add(new JLabel("Semestre:"));  pnlForm.add(cbSemestre);
        pnlForm.add(new JLabel("Correo:"));    pnlForm.add(txCorreo);

        // ── Panel de botones (FlowLayout alineado a la derecha) ──────
        JPanel pnlBotones = new JPanel(new FlowLayout(FlowLayout.RIGHT));
        btnLimpiar   = new JButton("Limpiar");
        btnRegistrar = new JButton("Registrar");
        pnlBotones.add(btnLimpiar);
        pnlBotones.add(btnRegistrar);

        // ── Barra de estado (South) ───────────────────────────────────
        lblEstado = new JLabel(" Formulario listo");
        lblEstado.setBorder(BorderFactory.createEtchedBorder());

        // ── Armar la ventana ──────────────────────────────────────────
        add(pnlForm,    BorderLayout.CENTER);
        add(pnlBotones, BorderLayout.SOUTH);
        add(lblEstado,  BorderLayout.NORTH);
    }

    private void registrarEventos() {
        // ActionListener con expresión lambda (Java 8+)
        btnRegistrar.addActionListener(e -> {
            String nombre = txNombre.getText().trim();
            String codigo = txCodigo.getText().trim();

            if (nombre.isEmpty() || codigo.isEmpty()) {
                JOptionPane.showMessageDialog(
                    this,
                    "Por favor completa todos los campos.",
                    "Campos requeridos",
                    JOptionPane.WARNING_MESSAGE
                );
                return;
            }
            lblEstado.setText(" ✔ Registrado: " + nombre + " [" + codigo + "]");
        });

        btnLimpiar.addActionListener(e -> {
            txNombre.setText("");
            txCodigo.setText("");
            txCorreo.setText("");
            cbSemestre.setSelectedIndex(0);
            lblEstado.setText(" Formulario limpiado");
            txNombre.requestFocus();
        });
    }

    public static void main(String[] args) {
        // Ejecutar la UI en el Event Dispatch Thread (EDT)
        SwingUtilities.invokeLater(() -> {
            new VentanaRegistro().setVisible(true);
        });
    }
}
🧵

Thread de UI: todas las actualizaciones a componentes Swing deben ejecutarse en el Event Dispatch Thread (EDT). Por eso se usa SwingUtilities.invokeLater() en el main. Operar la UI desde otros hilos puede causar errores difíciles de reproducir.

📝

Concepto clave — Herencia vs. Delegación: el ejemplo extiende JFrame directamente. Otra práctica común es crear una instancia de JFrame como campo de la clase (delegación). Para proyectos pequeños la herencia es más directa; para diseños más complejos, la delegación mejora la encapsulación.
