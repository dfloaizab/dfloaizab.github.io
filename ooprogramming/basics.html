Fundamentos de Programación Orientada a Objetos

Clase

Una clase es una estructura conceptual y técnica que describe las propiedades y comportamientos que tendrán los objetos que se creen a partir de ella. Una clase funciona como un molde o plantilla que define atributos (datos) y métodos (acciones).

En términos simples, una clase define cómo es algo y qué puede hacer.

Ejemplo conceptual

La idea de “Automóvil” puede considerarse una clase. Esa clase puede describir atributos como color, velocidad o marca, y comportamientos como acelerar o frenar.

Ejemplo en Java

class Automovil {

    String color;
    int velocidad;

    void acelerar(){
        velocidad += 10;
    }

    void frenar(){
        velocidad -= 10;
    }
}

Objeto

Un objeto es una instancia concreta de una clase. Si la clase es el molde conceptual, el objeto es la realización específica creada a partir de ese molde.

Los objetos poseen estado (valores concretos de atributos) y comportamiento (métodos definidos por la clase).

Ejemplo conceptual

Un automóvil rojo marca Toyota con velocidad de 60 km/h es un objeto específico derivado de la clase Automóvil.

Ejemplo en Java

public class Main {

    public static void main(String[] args){

        Automovil auto1 = new Automovil();

        auto1.color = "Rojo";
        auto1.velocidad = 60;

        auto1.acelerar();
    }
}

Diferencia entre clase y objeto

La clase define la estructura general y describe las propiedades comunes que tendrán todos los objetos de ese tipo. El objeto es una instancia concreta creada en memoria durante la ejecución del programa.

En analogía con la arquitectura, una clase equivale al plano de una casa, mientras que un objeto corresponde a una casa construida siguiendo ese plano.

Modificadores y palabras clave importantes

public

Indica que un atributo, método o clase puede ser accedido desde cualquier otra parte del programa.

private

Restringe el acceso únicamente a la propia clase. Se utiliza para proteger datos internos.

protected

Permite el acceso dentro del mismo paquete y también en clases hijas que heredan de la clase original.

static

Indica que un atributo o método pertenece a la clase en sí misma y no a instancias individuales. 

class Contador {

    static int total = 0;

    Contador(){
        total++;
    }

}

Encapsulamiento

El encapsulamiento consiste en ocultar los detalles internos de un objeto y permitir el acceso a sus datos únicamente a través de métodos controlados.

Ejemplos de la vida real

Un cajero automático permite retirar dinero mediante una interfaz simple, pero el usuario no tiene acceso directo al sistema interno del banco.

Un automóvil permite conducir mediante pedales y volante, pero el conductor no manipula directamente el funcionamiento interno del motor.

Ejemplo en Java

class CuentaBancaria {

    private double saldo;

    public void depositar(double cantidad){
        saldo += cantidad;
    }

    public double consultarSaldo(){
        return saldo;
    }
}

Abstracción

La abstracción consiste en representar únicamente las características esenciales de un objeto, ignorando los detalles irrelevantes para el contexto del problema.

Ejemplos de la vida real

Un control remoto permite operar un televisor mediante pocos botones, ocultando toda la complejidad electrónica interna.

Un mapa del metro muestra estaciones y conexiones relevantes, sin representar cada edificio o calle de la ciudad.

Ejemplo en Java

abstract class Animal {

    abstract void hacerSonido();

}

Herencia

La herencia permite que una clase derive de otra, reutilizando atributos y métodos. La clase derivada puede extender o modificar el comportamiento heredado.

Ejemplos de la vida real

Un perro y un gato pueden considerarse tipos específicos de animal.

Un profesor y un estudiante pueden considerarse tipos particulares de persona.

Ejemplo en Java

class Animal {

    void respirar(){
        System.out.println("Respirando...");
    }

}

class Perro extends Animal {

    void ladrar(){
        System.out.println("Guau");
    }

}

Polimorfismo

El polimorfismo permite que distintos objetos respondan de manera diferente a una misma operación.

Ejemplos de la vida real

El verbo “moverse” se manifiesta de forma distinta en un automóvil, una persona o un pez.

El método “emitir sonido” se expresa de manera distinta en un perro, un gato o un ave.

Ejemplo en Java

class Animal {

    void sonido(){
        System.out.println("Sonido genérico");
    }

}

class Perro extends Animal {

    @Override
    void sonido(){
        System.out.println("Guau");
    }

}

class Gato extends Animal {

    @Override
    void sonido(){
        System.out.println("Miau");
    }

}
